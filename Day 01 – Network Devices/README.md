
## Day 01 – Networking Devices

### Network Fundamentals
A computer network is a digital telecommunications system that enables nodes to share resources.

### Client–Server Model
- A client accesses a service.
- A server provides services for clients.
- A device may act as both client and server depending on the situation, such as in peer-to-peer networks.

### Switches (Layer 2)
- Provide connectivity within the same LAN.
- Offer many interfaces/ports for end hosts.
- Do not provide inter-LAN or Internet connectivity.

### Routers (Layer 3)
- Provide connectivity between LANs.
- Forward traffic over the Internet.
- Typically have fewer interfaces than switches.

### Firewalls (Layers 3, 4, and 7)
- Security devices that monitor and control traffic entering and exiting a network based on defined rules.
- Can be placed internally or externally in the network.
- Next‑Generation Firewalls include advanced filtering capabilities.
- Host‑based firewalls are software applications that filter traffic on individual machines.


### Repeaters – Extending Signal Distance (Layer 1 device)
- Repeaters strengthen a weak signal so it can travel farther.
- Regenerates electrical/optical signals
- Used in old Ethernet / long cables
  

### Bridges – The Predecessors of Switches(Layer 2 device)
- A bridge is an old switch with only 2–4 ports.
- Filters based on MAC addresses
Software‑based → slower
- Switches are the modern hardware‑based evolution

### Hubs – The Old, Dumb Devices(Layer 1 device)
- A hub is like a loudspeaker: when one device talks, everyone hears.
Why They Are Obsolete:
- They cause:
1. collisions
2. slow speed
3. no intelligence
- No MAC learning
- Broadcasts everything
- Switches replaced hubs almost completely.

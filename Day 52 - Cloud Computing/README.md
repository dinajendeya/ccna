
# **Day 52 - Cloud Computing**

## **1. What Is Cloud Computing?**

Cloud computing is the delivery of computing resources (servers, storage, databases, networking, and software) over the internet instead of running them on physical, on‑premise hardware.

In CCNA, you only need to understand the **core concepts**, not advanced cloud engineering.

***

## **2. Why Cloud Matters in CCNA**

Modern networks depend on cloud environments for:

*   Virtualization
*   Automation
*   Software-defined networking (SDN)
*   Remote and scalable infrastructure

Cisco expects CCNA students to recognize how traditional networks integrate with cloud-based systems.

***

## **3. Cloud Service Models (Required for CCNA)**

### **1️⃣ IaaS — Infrastructure as a Service**

You manage:

*   OS
*   Applications
*   Data

The provider supplies:

*   Virtual machines
*   Networking
*   Storage
*   Load balancers

Examples:

*   AWS EC2
*   Azure Virtual Machines
*   Google Compute Engine

***

### **2️⃣ PaaS — Platform as a Service**

You only focus on building applications.  
The provider manages:

*   Hardware
*   OS
*   Runtime

Examples:

*   Azure App Services
*   Google App Engine
*   AWS Elastic Beanstalk

***

### **3️⃣ SaaS — Software as a Service**

Fully managed applications delivered online.  
Examples:

*   Gmail
*   Microsoft 365
*   Slack

***

## **4. Cloud Deployment Models**

### **Public Cloud**

Shared infrastructure owned by providers like:

*   AWS
*   Azure
*   Google Cloud

### **Private Cloud**

A cloud environment dedicated to a single organization.  
Examples:

*   VMware vSphere
*   OpenStack

### **Hybrid Cloud**

A mix of private and public cloud, working together.

### **Multi‑Cloud**

Using multiple cloud providers at the same time.

***

## **5. Key Characteristics of Cloud Computing**

| Characteristic             | Description                                      |
| -------------------------- | ------------------------------------------------ |
| **On-demand self-service** | Users create resources without human interaction |
| **Broad network access**   | Accessible from anywhere via the internet        |
| **Resource pooling**       | Shared resources for efficiency                  |
| **Rapid elasticity**       | Scale up/down quickly                            |
| **Measured service**       | Pay only for what you use                        |

***

## **6. Cloud Networking Basics for CCNA**

### **Virtual Networks**

Cloud providers create virtual networks similar to VLANs in traditional networking.

### **Security Groups / Firewalls**

Act like cloud-based ACLs controlling inbound/outbound traffic.

### **Load Balancers**

Distribute traffic across multiple servers for performance and reliability.

### **VPN & Direct Connect**

Used to connect an enterprise network to the cloud:

*   Site-to-Site VPN
*   Dedicated connections

***

## **7. Benefits of Cloud Computing**

*   Reduced cost
*   Easy scalability
*   High availability
*   Disaster recovery
*   Fast deployment
*   Less need for physical hardware
